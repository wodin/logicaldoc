Module Hierarchy

  • translate.convert: translate.convert is part of the translate package It contains code to convert between different storage formats for localizations
    • translate.convert.convert: Handles converting of files between formats (used by translate.convert tools)
    • translate.convert.csv2po: convert Comma-Separated Value (.csv) files to Gettext PO localization files
    • translate.convert.csv2tbx: convert Comma-Separated Value (.csv) files to a TermBase eXchange (.tbx) glossary file
    • translate.convert.dtd2po: script to convert a mozilla .dtd UTF-8 localization format to a gettext .po localization file using the po and dtd modules, and the dtd2po convertor class which is in this module You can convert back to .dtd using po2dtd.py
    • translate.convert.html2po: convert HTML files to Gettext PO localization files
    • translate.convert.ini2po: convert .ini files to Gettext PO localization files
    • translate.convert.moz2po: convert Mozilla .dtd and .properties files to Gettext PO localization files
    • translate.convert.mozfunny2prop: converts funny mozilla files to properties files
    • translate.convert.odf2po: convert OpenDocument (ODF) files to Gettext PO localization files
    • translate.convert.oo2po: convert an OpenOffice.org (SDF) localization file to Gettext PO localization files
    • translate.convert.oo2xliff: convert an OpenOffice.org (SDF) localization file to XLIFF localization files
    • translate.convert.php2po: convert PHP localization files to Gettext PO localization files
    • translate.convert.po2csv: convert Gettext PO localization files to Comma-Separated Value (.csv) files
    • translate.convert.po2dtd: script that converts a .po file to a UTF-8 encoded .dtd file as used by mozilla either done using a template or just using the .po file
    • translate.convert.po2html: convert Gettext PO localization files to HTML files
    • translate.convert.po2ini: convert Gettext PO localization files to .ini files
    • translate.convert.po2moz: convert Gettext PO localization files to Mozilla .dtd and .properties files
    • translate.convert.po2oo: convert Gettext PO localization files to an OpenOffice.org (SDF) localization file
    • translate.convert.po2php: convert Gettext PO localization files to PHP localization files
    • translate.convert.po2prop: convert Gettext PO localization files to Java/Mozilla .properties files
    • translate.convert.po2rc: convert Gettext PO localization files to Windows Resource (.rc) files
    • translate.convert.po2tmx: convert Gettext PO localization files to a TMX (Translation Memory eXchange) file
    • translate.convert.po2ts: convert Gettext PO localization files to Qt Linguist (.ts) files
    • translate.convert.po2txt: convert Gettext PO localization files to plain text (.txt) files
    • translate.convert.po2wordfast: convert Gettext PO localization files to a Wordfast translation memory file
    • translate.convert.po2xliff: convert Gettext PO localization files to XLIFF localization files
    • translate.convert.poreplace: simple script to do replacements on translated strings inside po files
    • translate.convert.pot2po: convert Gettext PO templates (.pot) to PO localization files, preserving existing translations
    • translate.convert.prop2mozfunny: converts properties files back to funny mozilla files
    • translate.convert.prop2po: convert Java/Mozilla .properties files to Gettext PO localization files
    • translate.convert.rc2po: convert .rc files to Gettext PO localization files

  • translate.filters: Filters that can be used on translations...
    • translate.filters.autocorrect: a set of autocorrect functions that make resolving common problems more automatic...
    • translate.filters.checks: This is a set of validation checks that can be performed on translation units.
    • translate.filters.decoration: functions to get decorative/informative text out of strings...
    • translate.filters.helpers: a set of helper functions for filters...
    • translate.filters.pofilter: Perform quality checks on Gettext PO, XLIFF and TMX localization files
    • translate.filters.prefilters: This is a set of string filters that strings can be passed through before certain tests.
    • translate.filters.spelling: An API to provide spell checking for use in checks or elsewhere.

  • translate.misc: Miscellaneous modules for translate - including modules for backward compatibility with pre-2.3 versions of Python
    • translate.misc.autoencode: Supports a hybrid Unicode string that knows which encoding is preferable, and uses this when converting to a string.
    • translate.misc.dictutils: Implements a case-insensitive (on keys) dictionary and order-sensitive dictionary
    • translate.misc.ini: Access and/or modify INI files
    • translate.misc.multistring: Supports a hybrid Unicode string that can also have a list of alternate strings in the strings attribute
    • translate.misc.optrecurse
    • translate.misc.ourdom: module that provides modified DOM functionality for our needs
    • translate.misc.progressbar: progress bar utilities for reporting feedback on progress of application...
    • translate.misc.quote: string processing utilities for extracting strings with various kinds of delimiters
    • translate.misc.setup
    • translate.misc.sparse: simple parser / string tokenizer rather than returning a list of token types etc, we simple return a list of tokens...
    • translate.misc.stdiotell: A wrapper for sys.stdout etc that provides tell() for current position

    • translate.misc.wStringIO: A wrapper for cStringIO that provides more of the functions of StringIO at the speed of cStringIO
    • translate.misc.xmlwrapper: simpler wrapper to the elementtree XML parser
    • translate.misc.zipfileext: extensions to zipfile standard module that will hopefully get included in future...
  • translate.search: translate.search is part of the translate toolkit.
    • translate.search.lshtein: A class to calculate a similarity based on the Levenshtein distance.
    • translate.search.match: Class to perform translation memory matching from a store of translation units
    • translate.search.segment: Module to deal with different types and uses of segmentation
    • translate.search.terminology: A class that does terminology matching

  • translate.storage: translate.storage is part of the translate package It contains classes that represent various storage formats for localization
    • translate.storage.base: Base classes for storage interfaces.
    • translate.storage.benchmark
    • translate.storage.cpo: Classes that hold units of .po files (pounit) or entire files (pofile).
    • translate.storage.csvl10n: classes that hold units of comma-separated values (.csv) files (csvunit) or entire files (csvfile) for use with localisation
    • translate.storage.directory: This module provides functionality to work with directories.
    • translate.storage.dtd: classes that hold units of .dtd files (dtdunit) or entire files (dtdfile) these are specific .dtd files for localisation used by mozilla
    • translate.storage.factory: factory methods to build real storage objects that conform to base.py
    • translate.storage.html: module for parsing html files for translation
    • translate.storage.ini: Class that manages .ini files for translation
    • translate.storage.lisa: Parent class for LISA standards (TMX, TBX, XLIFF)
    • translate.storage.mo: Module for parsing Gettext .mo files for translation.
    • translate.storage.odf: This class implements the functionality for handling OpenDocument files.
    • translate.storage.oo: Classes that hold units of .oo files (oounit) or entire files (oofile).
    • translate.storage.php: classes that hold units of php localisation files (phpunit) or entire files (phpfile) these files are used in translating many PHP based applications
    • translate.storage.po: A class loader that will load C or Python implementations of the PO class depending on the use_implementation variable
    • translate.storage.pocommon
    • translate.storage.poheader: class that handles all header functions for a header in a po file
    • translate.storage.poxliff: An xliff file specifically suited for handling the po representation of xliff.
    • translate.storage.properties: classes that hold units of .properties files (propunit) or entire files (propfile) these files are used in translating Mozilla and other software
    • translate.storage.pypo: classes that hold units of .po files (pounit) or entire files (pofile) gettext-style .po (or .pot) files are used in translations for KDE et al (see kbabel)
    • translate.storage.qm: Module for parsing Qt .qm files
    • translate.storage.qph: Module for handling Qt Phrase Book (.qph) files.
    • translate.storage.rc: classes that hold units of .rc files (rcunit) or entire files (rcfile) these files are used in translating Windows Resources
    • translate.storage.statistics: Module to provide statistics and related functionality.
    • translate.storage.statsdb
    • translate.storage.tbx: module for handling TBX glossary files

  • translate.tools: translate.tools is part of the translate package It contains code to perform various operations (mostly on po files)
    • translate.tools.poclean: Produces a clean file from an unclean file (Trados/Wordfast) by stripping out the tw4win indicators.
    • translate.tools.pocompile: Compile XLIFF and Gettext PO localization files into Gettext MO (Machine Object) files
    • translate.tools.poconflicts: Conflict finder for Gettext PO localization files
    • translate.tools.pocount: Create string and word counts for supported localization files including: XLIFF, TMX, Gettex PO and MO, Qt .ts and .qm, Wordfast TM, etc
    • translate.tools.podebug: Insert debug messages into XLIFF and Gettex PO localization files
    • translate.tools.pogrep: Grep XLIFF, Gettext PO and TMX localization files
    • translate.tools.pomerge: Merges XLIFF and Gettext PO localization files
    • translate.tools.porestructure: Restructure Gettxt PO files produced by poconflicts into the original directory tree for merging using pomerge
    • translate.tools.posegment: Segment Gettext PO, XLIFF and TMX localization files at the sentence level
    • translate.tools.poswap: Builds a new translation file with the target of the input language as source language.
    • translate.tools.poterminology: reads a set of .po or .pot files to produce a pootle-terminology.pot
    • translate.tools.pydiff: diff tool like GNU diff, but lets you have special options that are useful in dealing with PO files
