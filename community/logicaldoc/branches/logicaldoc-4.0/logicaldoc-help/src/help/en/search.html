Search

In LogicalDOC you can search a document in four different ways:
1. Quick search
2. Advanced search
3. Parametric search (Enterprise Edition only)

Simple search


The simple search mask is always present and accessible any time in the upper part of the window.
It is important to specify the language in which the desired search has to be executed. As a start, the system proposes the default Document language for the customer (in English). By pressing on icon 3.2 system will search for the entered word.
Simple search will find only documents archived in the base language (and not documents in other languages).
If you want to search on documents written in any language, it is necessary to select the option "All documents".

Of every found element you can see the title, an extract of the containing text with the found words, the dimension in KBytes, a score and a link for:
  • Download the document (title of the document)
  • Select the document in folder (Show path -> click on the link path)
  • Show similar documents
  • Use the Document's Tools menu (Right Click on the title of the document)

The result presentation is shown in the form of a list ordered by the score reached.
The score is calculated on the frequency of the tried terms (or words with common root) of the text of the files.


In case of multiple results, the system will produce more pages than shown. The rest can be consulted pressing the cursor keys on the lower screen side to right of the results page.

Search on more words.
It is possible to search at the same time on more terms. Specifying, as an example, "future remote control" in the search field, there will become found, not only all documents that have the words indicated in the text, but also the documents that contain 2 of the 3 words or a single one of the 3 words.
We will see, besides, the documents that contain the words derived from "control": control, controlled, controller, those derived from "remote": remotelly and those which have future as root.

Use logical operators
In the simple search form it is already possible to use operators.
Using characters like "+ and -" it is possible to execute searches on obliged terms and using exclusion. As an example, composing the phrase "contracted +future -plan", the system will search all the documents that have the "future" word compulsorily and that compulsorily they do not contain the word "plan", while the "contracted" term comes inserted as optional.
A simple search for "contracted activity" instead, will find all the documents in which one of the 2 terms appears at least.
A search like "activity -contracted", will produce all the documents that contain the word "activity" and, at the same time, do not contain the "contracted" word.
Searching "+red +mario" is equivalent to red AND mario ". That means that we will try all the documents in which the bothd words red and mario appear.
It is also possible to explain these searches, as an example, "future AND project NOT development" is equivalent to "+future +project -development".
When there are used logical operators like AND, OR, NOT, it is necessary to keep in mind that they must be written compulsorily in upper case.

Advanced search


The mask for advanced search is accessible pressing the link Advanced Search, present to the right side of the simple search form in the uppper side of the window. The difference regarding the simple search is that such mask lets you specify, in a simple way, complex and active search criteria. Moreover, you can view the advanced search page under Searches menu.



This feature allows the user to search for one or more words through different search criteria. The search is performed on the contents and metadata of indexed documents and only on the search fields selected in checkbox. In addition, you can restrict your search only to a folder and, eventually, to its subfolders; you can search also among the attributes of a specific template (document type) associated with the searched document.
Using the advance search, it is possible to try simple words or phrases. Some words can moreover be excluded from the search or to activate an approximate search.
The result can be narrowed based on the dimension, the date and the format of documents.

Among the important aspects of such search:
  • the advance possibility of searching for terms that concur to express complex filters
  • searching for documents dimension
  • searching for document data (entered using the insertion/modification mask of the document)
  • searching for publication date (last modification of the document)

The result of this procedure of advanced search shows the same form as the one for simple searchs and the documents are accessible in the same ways.


Parametric search (Enterprise Edition only)

In LogicalDOC, under Searches menu you can view the parametric search page. This feature allows the user to search for documents that contain specific values on their metadata. The search is performed only on the metadata of the document and unlike the advanced search, all documents are taken into consideration, including those not already indexed. In particular, you can use the special character "%" to perform more sophisticated searches; for example, if you want to search for documents that have a custom ID with the string "invoice" at the beginning, then simply type "invoice%" into the "Custom ID" search field. If the searched string is contained in the custom ID, simply type "%invoice%" into the "Custom ID" search field.
