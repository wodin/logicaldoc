Getting Documents from mailbox

LogicalDOC has a way of recording email and attached files from remote mailboxes. It is possible to setup a number of mailboxes for the acquisition.
In regular timeintervals, the system will inspect these mailboxes to the search for new messages that will come automatically acquired and inserted into the archive as documents.

Mailboxes setup

You access the page management following the path: Administration --> Emails --> email accounts.


Centered on the screen, come visualized all mailboxes recorded into the system.
On first access, naturally this list will be empty.


In order to create a connection to a new mailbox, select the upper icon on the left side "Create E-mail Account".
The mask for the insertion of an email account appears to let you set up the normal parameters needed for the connection to a mail server.



The section "Storage" has all the necessary information for the treatment of messages in relation to archive of documents, in particular, and:
  • Allowed types contains a directory of extensions, separated by commas, that defines the single types of attachments that can be downloaded
  • Language sets up the language by which there will the messages be analyzed
  • Target folder is the folder in which the messages and their attachments will become saved as documents.

The new messages arrived from the mailbox will become downloaded and saved as documents in the set up folder.
The same treatment is assigned to the attachments (files attached in the email), but they will become archived as document sons of the main document that contains the body of the mail message.

Note:  
Only the new mails are downloaded, email downloaded previously is not downloaded again.
If one desires that the email remain on the server, check that the field "Delete from mailbox" is set up to "no".
You can schedule the import activity from mailbox through the "Scheduled Tasks".
