Folders

LogicalDOC places documents in folders. The idea is similar to what is being done in a local hard drive. Folders are logical collections of documents and are arranged in a hierarchical tree from the Documents folder. Organizing folders in an appropriate and logical manner will enable you to better navigate and find your documents.


Picture 9: A documents folder

Folders can be simply navigated through the navigation bar located on the left. There you will see and navigate the folders on which, you have the permission to view.


Picture 10: Folders navigation bar
 

Notice that a so-called breadcrumb (navigation path) is updated as you navigate through the folders and keeps track of your position. You can simply switch between folders using this path.


Picture 11: A folder's navigation path (breadcrumb)

Moreover, you can visualize the entire folder-tree by clicking on icon 1.2: LogicalDOC displays a window in which you can view all the folders
created by all users; by selecting one, you can see its content. This feature is very useful when there are plenty of folders and you want to
quickly switch from one to another.

Folder Creation

In LogicalDOC you can create a new folder by clicking on icon 1.1.
Next step is to give your folder a name and then click the Save button to complete the operation.


Picture 12: Folder creation
 

Note that LogicalDoc does not allow replicated names within the same folder. If you try to insert a folder named "mails" in the "Documents" folder, and there already exists a folder named “mails”, then the new folder will be automatically renamed to "mails(1)".

Editing folder name

In LogicalDOC you can rename a folder by clicking on icon 1.3. Then, you can change the name of the folder and click the Save button to complete the operation.

Permission setting

You can give permissions on folders to users and groups, which have been created in the system. You need to click on icon 1.4.


Picture 13: Permission setting
 


The permissions are related to:
  • Read
  • Write
  • Creation of a new folder
  • Permissions setting
  • Setting a document as immutable
  • Deletion
  • Editing folder name
  • Document import from zip archive
  • Document export to zip archive
  • Sign documents
  • Add documents to an existing archive
  • Setting workflow operations

The admin user and the admin group have all the permissions. By clicking on the bar Assign entities you can assign permissions on selected folder to users and groups. Moreover, you can move an entity from the ‘available list’ to the ‘assigned’ list by clicking on >>. By clicking on << you can bring the entity back to the ‘available list’. In order to select multiple entities, hold down the SHIFT key and click on the items you wish to select.

Searching documents in a folder

In LogicalDOC you can make an advanced search to look for documents inside a folder by clicking on the icon 1.5. Then LogicalDOC displays the advanced search page with the search already set on the current folder.


Picture 14: Search on documents in a folder

Export a folder in a zip archive

In LogicalDOC you can export the content of a folder by clicking on icon 1.6. Then LogicalDOC displays a window to save the entire content of the folder in a file called ZipExport.zip. This operation can only be done by users that have Bulk Export permission.


Picture 15: Export a folder in a zip archive

Folder deletion

In LogicalDOC, you can delete a folder by clicking on icon 1.7. Then a warning message appears to get your conformation for deletion. However,
if the folder contains immutable documents, LogicalDOC deletes all documents except immutable ones and will not remove the folder.
