Importing a .ZIP archive

Under some circumstances, it may prove to be useful allowing a number of documents contained in an specific folder organization to be inserted into the system on the fly, at once.
In LogicalDOC this functionality is available by importing from a ".ZIP" archive.
Thanks to this function, it is possible to replicate into the system the exact structure of "the zipped folder", either by the names and the nesting of folders or by the names and positioning of documents it contains.

It is necessary to keep in mind that there are, however, some limitations, the main of which is that the .ZIP archive to be uploaded must not be excessively large.
It is advised not to exceed the size 20 MB (megabytes).

First of all, the customer must create the folder with needed documents on his/her own PC.
Next, the customer must produce an ".ZIP" file of that folder, using a tool like WinZip, 7-Zip or similar.
Finally, created ".ZIP" file must be uploaded into LogicalDOC.

Upload ".ZIP" file

  1. In the Documents area, select the "father" folder that will accommodate the documents contained in the archive.
    The current path is shown on the right side of the screen.



  2. Clicking on the "Bulk Import" icon, it will appear a mask for the uploading of files similar to the one used for the insertion of a new document.
  3. Press the "Examine" key and select on the local hard disk the ".ZIP" archive.
  4. Press the "Upload" key in order to transfer the file into LogicalDOC.
    The progress bar indicates the advance of the uploading operation (file transfer).
    At the end, as the bar reaches 100%, it will appear the push-button "Bulk import (zip)" next to the lower right of the insertion mask.
  5. Press the push-button "Bulk import (zip)", in order to start the import.



The status pointer (up to the right) starts to flash signaling that an operation has been started.
After a while (that may be even of some minutes) becomes shown the contained folder with its new content.

Once started the operation, the customer must not abort it till be over. Since this could be of long duration, the session could be interrupted; in this case you do not have to worry, the system however, will proceed with the import; it will only be necessary to login again.

LogicalDOC will adopt the structure of the "zipped" folder and will copy all documents in the archive.
The new folder an all its documents will inherit all the permissions from the folder inside which they are imported.
